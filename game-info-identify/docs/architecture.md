# 架构设计

## 整体架构

```
┌─────────────────────────────────────────────────┐
│                    main.py                       │
│     流程编排 / 热键控制 / 模式分发 / JSON 输出     │
├──────────┬──────────┬───────────┬────────────────┤
│ capture  │ ocr      │ automation│ calibrate_tool │
│ 截图模块  │ 识别模块   │ 自动化模块  │ GUI 校准工具    │
└──────────┴──────────┴───────────┴────────────────┘
```

## 数据流

```
全屏截图 → 区域裁剪 → OCR 识别 → 稳定采样比对 → 收集到列表
                                              ↓
                              鼠标点击切换 → 下一账户

停止后 → 保存 data/YYYY-MM-DD_N.json → POST 到 sync.url（若已配置）
```

## 模块职责

| 模块 | 职责 | 关键依赖 |
|------|------|----------|
| `main.py` | 入口、流程编排、热键监听、JSON 输出 | 所有模块 |
| `capture.py` | 全屏截图、区域裁剪 | Pillow |
| `ocr_engine.py` | 图像预处理、数字/文字识别、稳定采样 | EasyOCR, Pillow |
| `automation.py` | 鼠标点击、Retina 缩放适配 | pyautogui, Pillow |
| `calibrate_tool.py` | 交互式截图框选、config 写入 | Tkinter, Pillow |

## 设计要点

### OCR 识别

`ocr_engine.py` 基于 EasyOCR 深度学习模型：

- 数字识别使用 `allowlist='0123456789'` 白名单限制
- 小文字自动 3× 放大提升识别率
- 宽图（宽高比 > 5）自动触发空间聚类，过滤图标等远端噪点
- Reader 懒加载单例，首次运行自动下载模型

### 稳定采样机制

`recognize_diamond_stable()` 采用连续两次比对+追加采样+多数投票的组合策略：

- 两次识别一致 → 直接采信
- 不一致 → 追加采样，直到连续两次一致
- 始终无法达成一致 → 多数投票，并保存调试截图

### Retina 屏幕适配

`automation.py` 通过比较截图物理像素和屏幕逻辑坐标，自动计算缩放比，确保在高分屏上点击位置准确。
